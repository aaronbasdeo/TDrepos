Breadcrumbs

Top-level Navigation > Feature Set > Narrower Domain > {variable} > Contextual Clue | [Button]

Navigational breadcrumbs in this guide are concise maps that help you to find and use platform features. In a documentation breadcrumb:

Boldface type cites a point of direct interaction, which you must click, toggle, or choose. In many cases, your action moves you from one page, tab, or modal popup to another. Alternatively, your action might toggle the visibility of other elements nearby, turn certain behaviors On or Off, or have another effect that is somehow essential to your path through the user interface. Boldfaced element labels — where an “element” is a menu or one of its options, a tab, a checkbox, or some other common point of interaction — often precede an angle bracket that is also boldfaced. Each such angle bracket signifies that there is a subsequent element to click, toggle, choose, or navigate past. For example:

     Go to This > That.     

Conversely, some labels within a documentation breadcrumb might not be set in boldface type. These are named elements that do not support direct interaction. Nonetheless, they help you to find other elements with which you must still interact. For illustrative purposes, suppose that a UI page features a menu labeled Options, a button labeled Options, and also a named area labeled Options, which includes a button that is labeled Configure. A breadcrumb that directed you to click that button could also, by not applying boldface formatting to its “Options” citation, direct you away from both the Options menu and the Options button — as follows:

     Go to This > That > Options | [Configure].     

Pipe characters — | — (as above) serve in place of angle brackets as a division within breadcrumbs when the subsequent element label or placeholder supports way-finding, rather than direct interaction.

Square brackets — [] — only ever offset the label that designates a button, which should be clicked. While the button label is boldfaced in a breadcrumb, its enclosing brackets are not. For example:

     Go to This > That > [Save].     

Curly braces — {} — draw your attention when an imprecise placeholder stands in for a precise label, which, in turn, may differ every time that you use it. (Suppose that the element to click is a username. However, precisely which username it is could vary each time that you navigate the path.) For example:

     Go to This > That > {username}.     

Parentheses — () — draw your attention to one named table in particular on a UI page that includes more than one named table. For example:

     Go to This > That > (Pending Events).     

Inline images — Beyond even the nuance that typographic signifiers can convey in a documentation breadcrumb, you might sometimes interact with user interface elements that are not labeled at all. In these cases, we describe the purpose or appearance in a general sense and show an inline image parenthetically. For example:

     Step 4.  Click the gear icon () to open a settings menu.     

OR

     Go to This > That | More, click the gear icon (), and then choose      
     Preferred Behavior from the settings list.
