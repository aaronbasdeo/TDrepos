Your marketplace logs all instances of nearly 50 different event types.

Three UI tables on the Event Logs page help you to find and interpret provisioning-focused events.

Integration Events — Features of the Integration Events table help you to browse, filter, and search all log entries that pertain directly to integrated products.

Pending Events — Features of the Pending Events table help you to browse, filter, and search all log entries about delayed or disrupted provisioning of products to their rightful users. Provisioning sometimes, does not occur as expected for products that should be provisioned automatically.

Microsoft SKU Transitions — The provisioning of Microsoft products is always asynchronous, meaning that each request to add or change user access is processed in hours, not seconds. Effects of asynchronous provisioning include that a subscription’s transition from one state to another is noticeably slow on Microsoft products and sometimes fails. Records in the Microsoft SKU Transitions table show you every Microsoft subscription whose state is pending.

Utilization

The last remaining event type that we log reveals exactly when and which registered users accessed exactly which products. You can browse, filter, and sort these logs from the App Usage Logs table.
