Download CSV (button)

Generates and downloads to your client system a CSV file whose records are the underlying basis for the Integration Events table as you filtered or otherwise shaped it.

Search (field)

Enter a text query string to run against all records in the table. Then, press Enter to submit the query.

Status (column)

One of these:

A green check mark icon signifies success.

A gray crescent wrench icon signifies that troubleshooting should occur.

Created on (column)

A calendar date and time stamp that states when the cited event was triggered.

Application (column)

The name of the product that provides context to the cited event.

Event Type/Event Token (column)

Stacks two values, one above the other.

The stacked value on top is the literal event type string that we sent to the product vendor, such as USER_ASSIGNMENT or SUBSCRIPTION_ORDER.

The stacked value below that is a hashed token that identifies the particular event verifiably.

User/Company (column)

Stacks two values, one above the other.

The stacked value on top is the full name of the user whose actions triggered the cited event.

The stacked value below that is the name of the organization whose associated user triggered the cited event.

Error Code/Message (column)

Stacks two values, one above the other.

The stacked value on top is the literal error type string that we sent to the product vendor, such as TRANSPORT_ERROR or OPERATION_CANCELED.

The stacked value below that is the verbatim error message, such as “Received error with message '404 Not Found' while calling integration endpoint.”

Manually Resolved? (column)

A boolean value (Y/N) that states whether any pending event in the corresponding record was manually marked as resolved.

Identifiers (account/user/generic) (column)

A 128-bit hash of the consumer’s identifying details.
