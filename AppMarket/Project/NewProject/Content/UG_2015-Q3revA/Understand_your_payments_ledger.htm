Understand your payments ledger

Navigation path

Manage > Channel > Marketplace > Billing | Payments

Your payments ledger shows an almost-live representation of payments as your billing system receives them. (The ledger does not refresh automatically in your browser, so its representation of payments is only as recent as when your browser loaded it.)

Overview

The overview-level view — via this toggled state above the table: — cites the combination of all successful and unsuccessful payments, excluding only those payments that applied a 100 percent discount. The Overview view renders seven columns in its UI version and 17 columns in its exportable CSV version.

The UI version is searchable against these values:

payment ID

customer

vendor

The UI version supports filtering by:

payment method

payments versus refunds

payment attempt outcome


Details

The detail-level view — via this toggled state above the table: — cites only the line items from successful payments. This excludes failed payments. The Details view renders eleven columns in its UI version and 35 columns in its exportable CSV version.

The UI version is searchable against these values:

payment ID

customer

vendor

The UI version supports filtering by:

payment method

payments versus refunds

invoice line type

whether a 100 percent discount was applied

when your marketplace processed the payment (or refund)
