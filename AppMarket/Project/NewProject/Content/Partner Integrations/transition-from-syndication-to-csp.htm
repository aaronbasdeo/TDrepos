Transition from Syndication to CSP

As part of our support for Microsoft resellers AppDirect works with Microsoft and Microsoft partners and assists them in the move from the current Syndication program to the new CSP program. Syndication is a legacy Microsoft reseller program that allows Microsoft Syndication partners to sell a variety of Microsoft products to their customers. It uses the Microsoft Online Syndication Interface (MOSI) set of Microsoft reseller APIs. The Cloud Solution Provider (CSP) is the reseller program that replaces Syndication. It uses the CREST Microsoft reseller APIs.

Warning: All customers must transition to CSP by the end of June 2017.

Prerequisites

  • The Dual Commerce feature should be enabled for the Marketplace.
  • All product profiles of deprecated Syndication SKUs should be hidden.
  • Each company that is going to be transitioned must be linked to a Syndication tenant.These companies must not have any deprecated Syndication SKUs. For more information please see the Microsoft table below.

Procedure

  1. Generate a dataset file. Select the Syndication customers you wish to transition. Go to Manage > Channel > Marketplace > Home | Companies.

  2. Search for the companies you would like to transition and click Download CSV to download the dataset file. If you require help please contact the AppDirect support team. This is an example of the MOSI CSV file:

  3. In the CSV file Name and Company are mandatory.
  4. To upload the dataset file go to Manage > Channel > Settings > Integration |Syndication to CSP transitions.
  5. In the Syndication customers section click Upload Dataset. Choose your dataset file and click Upload Dataset to upload your file. The same dataset can be uploaded multiple times.
  7. A batch entry is created with a status, a Microsoft transition input file and a report.
  8. View the Report. It contains the following fields:
    • Company ID​
    • Company Name
    • Microsoft Customer ID
    • Status
    • Error

  12. View the Microsoft Transition Input File. It contains the following fields:
    • Partner Name
    • Partner MSID
  14. The first 2 lines are the reseller account details. The rest of the lines contain information that Microsoft requires to transition the Syndication tenants and their subscriptions into the CSP. This is an example of transition input file for Microsoft:
  15. Send the Microsoft Transition Input File to Microsoft. This can be an Microsoft account executive who is working with you.
  16. Microsoft performs the transition and returns an Output File. This file contains details of the new CSP tenant and Subscription ID information. The Output File contains the following fields:
    • Customer Name
    • Customer Tenant ID
    • FpoCaid (Internal Microsoft customer identifier)
    • Mosi Customer ID
    • Mosi Subscription ID
    • CSP Subscription ID
    • CSP Subscription Offer ID
    • CSP Subscription Start Date
    • CSP Subscription End Date
  18. Upload the Output File. Go to Manage > Channel > Settings > Integration |Syndication to CSP transitions.
  19. In the Transitioned CSP customers and subscriptions section, select Upload Dataset.Choose the Output file your received from Microsoft and select Upload Dataset.
  20. View the report
  21. The report includes a list of all companies and subscriptions that were processed as well as a transition status value. It contains the following fields:
    • Company ID
    • Company Name
    • Domain
    • Old Subscription ID
    • New Subscription ID
    • CSP Offer ID
    • Transition Status
    • Error
    The AppDirect Marketplace validates the Old Subscription ID in the Output File. If no match is found, the status is marked as Failed.
  23. Once transitioned, the companies are linked to CSP tenants and subscriptions.
