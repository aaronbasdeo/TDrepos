Access a password-protected server

AppDirect-powered marketplaces support the option, via a UI control that adjusts httpd.conf on the backend, to password-protect everything on their host server, including even its standard login form for individual users.

This feature is used most commonly when a marketplace is not yet open for business, while it undergoes basic testing. It prevents user dissatisfaction with a site that is reachable but not actually ready for use. However, your relationship with a given marketplace — maybe as one of its administrators or developers — might require your engagement with it long before ordinary consumers learn that the server even exists.

In such cases (or whenever else this option is used), you must know the sitewide credentials that compel the server to send page data to your browser. These credentials almost certainly differ from your personal username and your personal password, which you must then enter as usual.

Most marketplaces are not secured in this way.

Procedure

Go to https://{marketplace}, where {marketplace} is the fully qualified and DNS resolvable server name for your marketplace instance.

Your browser does not receive any data to render the usual user interface for your marketplace. Instead, a modal dialog box opens, which prompts you to enter sitewide credentials.

Enter the sitewide username and the sitewide password. Then, press Enter.

When you submit valid sitewide credentials, the modal dialog box closes and the server returns enough information that your browser can render the marketplace user interface — including the standard login form for individual users.

Stop. You have completed this procedure.
