Add add-on offering

Add-on offerings are additional items or services that customers can purchase when they buy a product. For example, an add-on might include a preset amount of additional storage space or time-limited access to a professional service.

Every add-on is specific to a product, and every web application or stacked web application product can have one or more add-ons. No other product types support add-ons.

An add-on cannot be purchased on its own. It is only after adding an associated product to an order that a purchaser sees that an add-on is available. Add-ons are always optional.

User management (assignment and unassignment of users) applies to a product and its add-ons as a whole. An add-on cannot be assigned to or unassigned from a user independently of the product.

An add-on can be canceled from an order without canceling the product or other add-ons. However, when an order includes a product and an add-on, the add-on must be canceled before the product can be canceled.

Add-ons are never required. They are always optional but require that the customer have or purchase an edition along with the add-on. 

To add an add-on offering

  1. Navigate to Developer > Dashboard | Products | {Product} > Edit. This opens the product configuration page. On the left side of the page, you can see the number of add-ons currently available for this product. In the following example, the number is 0, indicating that no add-ons have been created.

  2. Click Add-on Offerings to open the Add New Add-On Offering button.

  3. Click Add New Add-on Offering. This opens the Create New Add-on page.

  4. Complete the fields on the page and select the options as appropriate for your add-on. Following is a description of each field and option.  

  5. Add-on Position—If you create more than one add-on, the number you select here determines the order in which this add-on is displayed on the product profile page. 

  6. Name—This is the primary heading for the add-on. 

  7. Code—The code is used to identify this specific add-on for the notification URLs. It does not need to be unique on the marketplace but it does need to be unique within your product. If not applicable, enter any value.

  8. Short Description—Creates a five to ten word description that is displayed on the Pricing & Editions page of the product profile and on the Create Order page for checkout. 

  9. Detailed Description—This short paragraph description is displayed on the Create Order page of the profile. 

  11. Charge Frequency—The frequency at which the user is charged for the product, as follows.

    • One time fee—One time add-ons charge users a flat amount once. Customer always have the add-on unless they intentionally decide to delete it. 

    • One time per unit—One time add-ons charge the user once for a set quantity of units. The price per unit is multiplied by the number of units selected. Customers always have the add-on unless they intentionally decide to delete it. 

    • Monthly fee—A monthly fee add-on results in the user being charged a flat amount per month. The add-on can be canceled at any time. 

    • Monthly per unit—A monthly per unit add-on results in the customer being charged for a number of units per month. The price price per unit is multiplied by the number of units selected. The units can be canceled at any time. 

    • Monthly metered—A monthly metered add-on means the customer is charged based on usage for a particular unit over a month. The price price per unit is multiplied by the number of units reported by the developer, as used by the customer. The customer can cancel at any time. 

  12. Allow stacked purchases—When you check this box, it allows the same add-on to be purchased multiple times. By returning to the checkout flow, the customer can buy another version of the add-on. The customer now sees two separate add-ons with the same name under Manage Apps. 

  13. The stacked purchase option can be applied to the following frequency options:

  14. One time fee

  15. One time fee per unit

  16. Monthly fee

  17. Exclusive add-on—An exclusive add-on cannot be purchased along with any other add-on. 

  19. Price Rate—The price you enter is what the customer is charged, at the frequency determined under Charge Frequency. If your marketplace is configured with more than one currency, each currency is displayed under the Price Rate area, and you must enter the corresponding rate for each currency. This would typically apply when your service is sold in multiple countries and users are charged in their own currency.

  21. Feature—Adding one or more features allows you to provide information to end users about why they would want to choose this add-on. The feature is created in the form of a bullet point.

  22. Bullet point—Enter one bullet point per feature. Short sentences are best to allow the customer to quickly compare editions

  23. Bullet tool tip—Adding a bullet tool tip provides additional information when a user hovers over the bullet point.

  24. To add more features (bullet points), click Add another feature.

  25. Step 5. After all fields and options are entered, you can:

  26. Save and preview the add-on

  27. Click Save Add-on to save it

  28. Click Delete Add-on to delete the add-on

  29. After you save the add-on, you can add more add-ons in the same manner.
