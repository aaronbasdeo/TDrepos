After a Developer creates a product, they can create product versions. This is required in the following situations:

  • You need to create an updated version of the downloable file (for example, for a bug fix, or to introduce a new feature).
  • You want to expand the set of devices that your product supports. The new product version has different compatibility attributes (for example, a new operating system version).

The following procedure outlines the steps to create a new product version.

Procedure

Step 1. Ensure that AppDirect has enabled and configured platforms for your marketplace.

Step 2. Go to Manage > Billing and Distribution > Products > {name of product} > Edit > Devices > Platforms. The list of existing platforms displays in the left navigation in the Devices pane.

Step 3. Click the name of the platform where the product for which you want to add a new version resides.

In this example, the platform is Microcomputers. Clicking the platform name opens the Product Versions page, with a list of existing product versions. Each product version lists the version name, version code, status and when it was last updated.

There are two ways to create a new product version :

  • Duplicate the version you are viewing. Use this option when the new product version has many of the same configurations as the existing one. This allows you to create a new product version more rapidly. Change only the details required to create the new product version, and upload the correct asset file for the new product version.
  • Click Add Product Version. Proceed to step 4 for this option.

Step 4. Click Add Product Version. The product version creation page opens.

  • Note that the top section is incomplete, since the product version is in draft form.

Step 5. Enter all required information in the Product Version Details pane, following these procedures:

a. Title—Internal name used in the content management system for the Product Version. Allows the Marketplace Manager to quickly differentiate between product versions. This name is not visible to customers.

b. Version Code—Used to determine whether one version is more recent than another. This must be a positive integer that is higher than the value entered for the previous product version. If left blank, the next incremental value is entered automatically when the product version is saved. 

c. Version Name—The product version name that is visible to users on their devices. Change the name that automatically appears in this field to something that is meaningful for customers.  For example: Initial free version.

d. File Asset—Click Choose File to upload the file that will be downloaded to the device. Each product version has its own asset file.

e. Product version-specific fields—Enter required information in each of the remaining fields in the Product Version Details pane. When creating the platform, the Marketplace Manager defined the information that must be configured to create a product version. These can include text fields, checkboxes or drop-down lists.  For example: Model, operating system, RAM, required connectivity, and others.

Step 5. Click Save. A notification appears, confirming that the product version has been saved. Additionally, fields that were previously gray and empty are now completed. You can make further edits if required at this stage, then click Save again.

Step 6. Click Request Approval. This notifies the Marketplace Manager that the product version is ready for their approval. The Marketplace Manager approves and pubishes the product version. If the request is denied, you receive a notification with an explanatory message. In that case, make the required changes using the procedures previously described, and click Request Approval again.

After you click Request Approval, you are returned to the Product Versions Detail page, where you can take other actions to manage product versions.
