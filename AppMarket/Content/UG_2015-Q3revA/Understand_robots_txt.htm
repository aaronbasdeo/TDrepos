Understand robots.txt

Navigation path

Manage > Channel > Settings > Settings | Marketplace Functionality

Search engine webcrawlers use a website’s robots.txt file to learn which parts of a site should be indexed and which parts should not. You can edit the robots.txt file for your marketplace. Its logic should be parallel with these examples:

 NOTE:  Replace {spider_name*} in these examples with the actual name of the search engine spider whose behavior you mean to constrain, such as googlebot, bingbot, or slurp.

All spiders can index everything

User-agent: *

Allow: /

All spiders are forbidden to index anything

User-agent: *

Disallow: /

All spiders are forbidden to index specified subdirectories

User-agent: *

Disallow: /example1/

Disallow: /example2/

Disallow: /example3/

All spiders are forbidden to index a specified file

User-agent: *

Disallow: /example/examplefile.html

A specified spider is forbidden to index anything

User-agent: {spider_name}  

Disallow: /

Two specified spiders are forbidden to index a specified subdirectory

User-agent: {spider_name1}  

User-agent: {spider_name2}  

Disallow: /example/

Differing permissions for different spiders

User-agent: {spider_name1}  

Allow: /example1/

 

User-agent: {spider_name2}  

Disallow: /example2/
