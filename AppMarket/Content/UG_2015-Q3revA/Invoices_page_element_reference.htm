Invoices page element reference

Navigation path

UI fields and elements on the Invoices page

Element

Description

Show Filters (button)

Exposes a banner above the table, where UI elements can help you to shape which records the table should represent — both in its visible rows and, optionally, in any CSV file that you generate and download.

 

Status (list) — One of the following. Or Show All, the full combination of all the following.

 

Paid — Invoices for which your marketplace has record of payment.

 

Unpaid — Invoices for which your marketplace has no record of payment.

 

Carried — Invoices for which the amount due is deferred to a subsequent invoice.

 

Created between (date picker) — Define the start and end of a date range by entering dates manually in the paired date-entry fields, using the notation syntax that is standard in your locale and which your marketplace is configured to use — either MDYY or DMYY. Alternatively, click a field’s corresponding calendar icon () to expose the date-picker widget for populating that field.

Download CSV (button)

Generates and downloads to your client system a CSV file whose records are the underlying basis for the Invoices table as you filtered or otherwise shaped it.

Search (field)

Enter a text query string to run against all records in the table. Then, press Enter to submit the query.

ID (column)

Cites a unique, numeric identifier, often six digits long, that designates the described invoice.

Date (column)

Cites the invoice generation date.

Customer (column)

Cites the invoiced contract’s billable company by name.

Total (column)

Cites the total amount due on the described invoice. This amount might include amounts from one or more earlier-dated invoices, whose balance was carried forward.

Status (column)

x

(pagination controls)

Click these UI elements under the table to jump between respectively distant ranges of records in it, when it contains more records than it is configured to render at one time as visible rows.
