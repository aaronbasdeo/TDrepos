The sequence of transitions in Scenario 1

The chronology of transitions in this scenario is as follows.

Order and subscription state transitions in Scenario 1

 

A newly placed order starts its life in the INITIALIZED state when all of these statements are true simultaneously:

The order specifies a hosted product whose vendor must provision it.

The order should take effect immediately, and therefore is not post-dated.

The order does not apply any discount.

Alternatively, the purchase order for a billable and auto-renewable product edition can begin in the INITIALIZED state (upon contract term renewal) when either of the following statements is true:

A channel administrator has blocked in-term contract changes for all products, sitewide.

The product vendor has blocked in-term contract changes for this product in particular.

There is no subscription state at this stage. A subscription does not yet exist.

 

Transition to a PENDING REMOTE CREATION state occurs when your marketplace notifies the product vendor that an order was placed, which requires provisioning. (Here, the requested edition is billable.)

The subscription state is INITIALIZED.

 

Transition to an ACTIVE state occurs when the relevant vendor finishes provisioning. The consumer is now truly subscribed to a billable product that carries a recurring billing cycle.

The subscription state is ACTIVE.

 

Transition to a CANCELED state occurs (manually in this case) when the consumer requests to terminate the subscription that a given order covers (rather than merely updating the subscribed edition or units). However, stipulations of the corresponding subscription contract might:

Mandate that the subscription must continue, uninterrupted, through a minimum stipulated number of billing cycles.

Impose penalties for early cancellation.

Therefore, cancellation itself might take effect after a future billing cycle concludes.

The subscription state is CANCELED.
