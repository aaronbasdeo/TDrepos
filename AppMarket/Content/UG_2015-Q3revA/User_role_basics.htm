User role basics

An AppDirect-powered marketplace serves the overlapping needs of its diverse population of consumers, developers, resellers, providers, and others. We subdivide these broad user categories — which also designate the various company types — into multiple, discrete user roles, which are assignable to registered users exclusively. Users who all share a given role assignment in common also share certain expectations, needs, and goals in common. AppDirect supports a range of user roles for a range of purposes.

 TIP:  This guide consists of related books that focus on the goals and authority of one user role apiece. Channel administrators can learn about company and user account management in the Marketplace Manager book chapter called Customers.

What is a visitor?

When this guide touches upon the superficial access that some marketplaces allow to unauthenticated users — including those who are unauthenticated precisely because they have not registered any user account — it calls these people “visitors.” This pseudo-role is implicit and conditional. It exists on some marketplaces but not others and is never explicitly assigned.

All other user roles assignments are explicit and active, meaning that registered users receive them through an administrator’s deliberate intervention.

User role variations

User roles and their names are somewhat variable from one marketplace to another, because some roles are not relevant to some business plans and preferred terminology can differ regionally or by industry. Furthermore, some roles must be enabled per marketplace before they are assignable to any user, or must be enabled per company before they are assignable to a given company's associated users.

User role names and descriptions in this guide are generic and might differ from user roles as your marketplace implements them.

Certain potentially available user roles on a given marketplace:

Might be restricted to some plans in particular.

Might be disabled at the provider’s request and unavailable for assignment.

Are only ever enabled upon the provider’s request to AppDirect.

Are available for assignment only under narrow conditions.

Are enabled by default but can be disabled nonetheless.

Additional limitations may also apply, as follows.

The UI labels that specify user role names can be changed per marketplace, upon the provider’s request to AppDirect.

Some marketplaces implement additional, nonstandard user roles.

The user roles that are assigned to you constrain which other user roles you can see, assign, and manage, if any.
