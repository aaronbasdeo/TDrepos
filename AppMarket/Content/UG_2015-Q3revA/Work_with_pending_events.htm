Work with pending events

Navigation path

Manage > Channel > Marketplace > Events | Event Logs > (Pending Events table)

Before you begin

Log in to your marketplace as its administrator.

Procedure

Go to Manage > Channel > Marketplace.

Click through Events | Event Logs. Then, scroll to the Pending Events table.

(Optional) Click Show Filters to expand a toolbar whose controls help you to filter which event records the table should show. Then, do any or all of the following.

 NOTE:  Filter selections take effect immediately, and each effect is additive.

 

Do you want to see log entries from a specific time frame?

(Optional) Filter events by when they occurred

Define the start and end of a date range by entering dates manually in the paired date-entry fields.

OR

Click a field’s corresponding calendar icon () to expose the date-picker widget for populating that field.

 NOTE:  When you enter dates manually, use the calendar date notation syntax that is standard in your locale and which your marketplace is configured to use — either M/D/YY) or D/M/YY.

Do you want to limit the table to list events of only one type?

Set just one event type to list

Choose an option from the Event Type list.

(Optional) Do any of the following:

Click a name in the Creator column to view details about the corresponding user account.

Click a name in the Company column to view details about the corresponding organization.

Click a name in the Application column to view details about the corresponding product.

Click the Resolve button in a given table row to open a modal popup where you can view details about the failed or stalled request that the same row cites and, then, either resubmit the request for processing or mark the event as manually resolved.

Stop. You have completed this procedure.
