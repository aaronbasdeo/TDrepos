The sequence of transitions in Scenario 7

The chronology of transitions in this scenario is as follows.

Order and subscription state transitions in Scenario 7

 

A newly placed order starts its life in the INITIALIZED state when all of these statements are true simultaneously:

The order specifies a hosted product whose vendor must provision it.

The order should take effect immediately, and therefore is not post-dated.

The order does not apply any discount.

There is no subscription state at this stage. A subscription does not yet exist.

 

Transition to a PENDING REMOTE CREATION state occurs when your marketplace notifies the product vendor that an order was placed, which requires provisioning. (Here, the requested edition is a free trial.)

The subscription state is INITIALIZED.

 

Transition to a FREE TRIAL state occurs when the vendor finishes provisioning the specified product’s free trial edition.

The subscription state is FREE TRIAL.

 

Then, in this case, transition to an ACTIVE state occurs:

After the consumer updates his or her entitlement from the free edition to a paid edition.

When the vendor finishes provisioning one of the same product’s billable editions.

The subscription state is ACTIVE.

 

Finally, transition to a CANCELED state occurs (manually in this case) when the consumer cancels the subscription.

The subscription state is CANCELED.
