Work with migration data

Before you begin

Pay for the AppDirect migration service.

When ad-blocking or popup-blocking software is installed on your client system, add your marketplace server to the whitelist.

Log in to your marketplace as its administrator.

Navigation path

Manage > Channel > Settings > Integration | Migration

Procedure

Go to Manage > Channel > Settings.

Click past Integrations | Migration.

Do one of the following.

Do you want to create a migration dataset?

Create a dataset

Click Create Dataset.

The Data Migration page opens.

(Optional) Click Download to obtain a sample migration data file, which you can examine to learn about structure and syntax conventions for each type of migratable record. [The button is now missing. See MIG-630 in JIRA.]

Enter a name in the Batch Name field, which pertains correctly and equally to each of the migration files you plan to upload in this session. You must enter a name.

There are multiple upload forms — one apiece for each of the record types that you can migrate.

Users

Companies

Memberships

Reseller Associations

Discount Codes

Company Entitlements (meaning, subscription states}) [per Mary, the word “entitlement” should never be external]

User Entitlements (meaning, seat assignments) [per Mary, the word “entitlement” should never be external]

Microsoft Customers

Microsoft Subscriptions

Microsoft Domains

Google Accounts

Do the following.

Click Choose File in each form that corresponds to data which you have processed and ready for migration.

When the File Upload window opens, browse through your client system’s mounted volumes and folder structure until you find the appropriate file for a given form.

Click the data file to select it, and then click Open.

Repeat these steps as many times as needed to complete your dataset.

Click Upload Dataset.

Do you want to review a migration report?

Review a migration report

Browse through the Data Migration table until you locate the dataset whose migration details matter to you.

Click Download Report in the same table row that cites the dataset’s name.

Open the report in your preferred spreadsheet software and check any entries in the Errors column.

Do you want to download product login credentials?

Download product credentials

Browse through the Data Migration table until you locate the dataset whose migration details matter to you.

Click Download Credentials in the same table row that cites the dataset’s name.

Open the credentials file in your preferred spreadsheet software to review it.

Stop. You have completed this procedure.
