The sequence of transitions in Scenario 23

The chronology of transitions in this scenario is as follows.

Order and subscription state transitions in Scenario 23

 

A newly placed order starts its life in the PENDING USER APPROVAL state when a reseller (or other authorized representative of your marketplace) has placed an order on behalf of a registered user.

There is no subscription state at this stage. A subscription does not yet exist.

 

Transition to a PENDING REMOTE CREATION state occurs after the registered user consents to this assisted purchase — and after your marketplace notifies the product vendor that an order was placed, which requires provisioning (and here, the requested edition could be either billable or a free trial version) — when all of these statements are true simultaneously:

The order specifies a hosted product whose vendor must provision it.

The order should take effect immediately, and therefore is not post-dated.

The order does not apply any discount.

The subscription state is INITIALIZED.

 

However, transition to the FAILED state occurs when the product vendor is not able to provision the ordered product. We recommend that you investigate the basis for this failure and work with the vendor to resolve it — both to benefit the affected consumer right now and so that no similar failures affect your other consumers in the future.

The subscription state is FAILED.
