Configure product discounts

You can x

Navigation path

Manage > Channel > Products > Promotions | Discounts

Procedure

Go to Manage > Channel > Products.

Click past Promotions | Discounts.

Click Add Discount.

Set the redemption method.

Click Apply automatically when your marketplace should redeem the discount without consumer action for each order toward which it is valid.

OR

Click Enter code at checkout when consumers should enter the discount code manually to apply it toward an order, or even to check if it can be applied to the order.

When you enforce manual entry of discount codes during checkout, three more options are exposed, from which you must choose one.

Enter code at checkout — Exposes a text-entry field when selected. Enter exactly the code in this field that buyers must also enter manually during checkout to apply this discount to an order.

Generate multiple random codes — Exposes the Number of random codes box, where you can set any number in the range from 1 to 999. Later, you can distribute these codes one at a time to any buyers who should know them.

Upload a file containing your discount codes — Exposes the Upload File selector. Click Choose File to start navigating through your client system’s mounted volumes and their respective folder structures. When you locate the text file that should be uploaded, click it to select it. Then, click Open to upload it.

NOTE:  What we call the Open button here could have a different label in your browser. This button label is determined not by your marketplace but by your browser.

Do one of the following to set the discount.

Click Percentage when the discount should amount to an equal proportion of each order that redeems it — such as 10 percent toward any order. Then, enter the percentile integer in the Percentage Discount field.

Click Fixed price when the discount should have an absolute value toward each order, which never varies, such as exactly $1.00 toward any order. Then, enter the absolute sum in the Fixed Price Discount field. The currency that applies to your entry here is whichever currency your marketplace uses by default.

Configure discount details.

Enter a brief summary of the discount’s purpose in the Details field.

(Optional) Use the Discount Logo form to upload a logo for the discount itself.

Choose an option from the Product list. Supported options are Apply to All and the name of every published product.

Choose an option from the Edition list. Supported options vary according to your choice from the Product list.

Choose an option from the Payment Plan list. Supported options vary according to your choice from the Edition list. [We have always claimed that the terms Edition and Payment Plan were perfectly synonymous. But in this UI, without any clarification, they are not. What changed our reasoning? Or is this simply an error?]

Set any redemption restrictions.

Click No restrictions per user when any number of qualifying users are able to redeem the discount any number of times.

Click A user can redeem this discount only once when each qualifying user is able to redeem the discount one time.

Click Member of a company can redeem this discount only once when only one qualifying user per company context — the first user to apply it — is able to redeem the discount one time.

Enter an integer in the Max Redemptions field to limit how many orders can redeem the discount.

Set a date range for when the discount should be active.

Use the Start Date calendar tool to open a discount validity date range.

Use the Expiration Date calendar tool to close a discount validity date range.

Enter an integer in the Billing Cycles field to limit how many billing cycles per order should apply the discount.

Click Save.

Stop. You have completed this procedure.
